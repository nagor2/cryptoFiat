## Варианты завершения перевода

### Вариант 1: Использовать ChatGPT/Claude (РЕКОМЕНДУЕТСЯ)

**Самый быстрый и качественный способ**

1. Открой ChatGPT или Claude
2. Для каждого файла используй промпт:

```
Переведи этот документ с английского на русский, сохраняя:
- Всю структуру RST разметки
- Все заголовки и форматирование
- Все ссылки и коды
- Технические термины (TVL, CDP, DeFi оставь как есть)
- Профессиональный, но понятный стиль

[вставить содержимое файла]
```

3. Скопируй перевод в соответствующий файл
4. Проверь форматирование

**Время**: ~2-3 часа на все файлы

### Вариант 2: Автоматический перевод + ручная правка

1. Используй Google Translate API или DeepL API
2. Автоматически переведи все файлы
3. Вручную исправь технические термины и форматирование

**Скрипт для автоматизации:**

```bash
# Установи DeepL CLI
pip install deepl

# Переведи все файлы
for file in ../docs/pages/*.rst; do
    filename=$(basename "$file")
    deepl translate --from en --to ru "$file" > "pages/${filename}"
done
```

**Время**: ~4-5 часов (включая правку)

### Вариант 3: Ручной перевод

1. Открывай каждый файл в редакторе
2. Переводи секция за секцией
3. Сохраняй форматирование RST

**Время**: ~20-30 часов

**Не рекомендуется** - слишком долго

### Вариант 4: Нанять переводчика

1. Найди технического переводчика с знанием крипто
2. Стоимость: ~$0.05-0.10 за слово
3. Бюджет: $2,000-4,000 для 40,000 слов

**Плюсы**: Профессиональное качество
**Минусы**: Дорого, занимает время

## Советы по переводу

### Что НЕ переводить:
- TVL, CDP, DeFi, DAO, CEX, DEX, APY, NFT
- Названия протоколов (Uniswap, Aave, etc.)
- Технические термины (smart contracts можно оставить)
- URLs и ссылки
- Блоки кода

### Что переводить:
- Весь текст и объяснения
- Заголовки (но технические термины оставь)
- Комментарии в коде
- Примеры и use cases

### Сохранение форматирования RST:
```rst
.. важно:: Сохрани все директивы RST как есть

Заголовки
=========

Подзаголовки
~~~~~~~~~~~~

- Списки
- Тоже сохрани

::

    Блоки кода не трогай
```

## Проверка качества

После перевода проверь:

1. **Форматирование**
   ```bash
   cd docs-ru
   make html
   # Не должно быть ошибок
   ```

2. **Ссылки**
   - Все внутренние ссылки работают
   - Относительные пути правильные

3. **Читабельность**
   - Текст звучит естественно по-русски
   - Нет машинного перевода
   - Технические термины используются правильно

4. **Полнота**
   - Весь контент переведен
   - Ничего не пропущено
   - Структура сохранена
